# ふくおかポーカーナビ（福岡ポーカートーナメント日程アグリゲーター）

福岡県内のアミューズメントポーカー店のトーナメント日程を一括表示する公開サイト。
エースハイ合同会社の収益プロジェクト v1。

## これは何か / 収益設計

- **資産になる「サイト」を母艦**にする（SEO：「福岡 ポーカー トーナメント 日程」等）。
- **LINE公式 / X**は無料の通知・回遊チャネルとして**サイトへ送客**（収益はサイト側）。
- 収益スロットは3層を最初から埋め込み済み：
  1. **広告（AdSense）** … `.adslot` の位置。PVが付けば自動で回る。
  2. **店舗のPR/有料掲載** … `venue.featured = true` でカード上部固定＋「PR」バッジ＋★。単価が高い本命。
  3. **周辺商品アフィリ** … 記事/サイドに随時追加（今後）。

## ファイル

| ファイル | 役割 |
|---|---|
| `index.html` | 公開サイト本体（モバイルファースト・単一HTML） |
| `admin.html` | 管理コンソール（店舗を選ぶ→画像/テキストを渡す→登録するだけの1画面） |
| `dev-server.py` | 管理コンソール用のローカルサーバー（`/save-data` で `data.js` に自動保存） |
| `data.js` | 表示用の正規化済みデータ（`VENUES` / `TOURNAMENTS` / `AREAS`）とスキーマ定義 |
| `jopt-data.js` | 大型イベント JOPT 2026 Fukuoka #01 のトーナメント・ストラクチャーデータ（約116KB）。**`#jopt` を開いたときにだけ動的読み込みされる**（全ページで読むには重いため）。`index.html` の `JOPT_META`（日程・件数）と二重管理になっており、日程を差し替える際は両方の更新が必要（件数・日程のズレはデータ読み込み時に自動訂正されるが、ホーム画面のバナー表示判定だけは `JOPT_META` の値で行われる） |
| `fukuoka-venues.json` | 実在店の調査結果（出典・注記付き。`data.js` の VENUES の原典） |

`data.js` を差し替えるだけでサイトが更新される設計。CDN/静的ホスティング（GitHub Pages 等）にそのまま置ける。

## 管理コンソールの使い方（月末の更新作業）

```
cd fukuoka-poker
python3 dev-server.py
```
と実行し、表示された `http://localhost:8787/admin.html` を開く。

1. 上部の「対象月」を確認・変更する（翌月分を入力する場合は翌月に変更）。
2. 店舗一覧から未入力の店舗（一覧の上側）をクリックして選ぶ。情報取得先のリンクがあればそこから告知画像を確認できる（未設定ならその場で「+ リンク追加」から登録できる）。
3. 告知画像をドラッグ&ドロップ／貼り付け（⌘V）／ファイル選択で追加する（テキストしかない場合は「画像がない場合は代わりにテキストを貼り付け」を開いて貼る）。
4. 「この店舗として登録する」を押す。抽出→登録までこの1クリックで完了し、変更は自動で `data.js` に保存される（ブラウザの操作だけでよく、ダウンロード→手動コピーは不要）。
5. 登録済みの店舗は一覧の下側（✓済み）に移動する。全店舗が済みになるまで2〜4を繰り返す。

Claude APIキーは初回だけ「詳細設定」で登録すればよい（このブラウザにのみ保存）。`dev-server.py` を使わずに `admin.html` を直接開いた場合（GitHub Pages上など）は自動保存が効かないため、「詳細設定→手動エクスポート」からダウンロードして手動で差し替える。

## データ取得アーキテクチャ（自動化の現実解）

「完全自動」はソースによって可否が分かれる。**取れるものは自動、取れないものは半自動**のハイブリッドにする。

| ソース | 自動化 | 方法 |
|---|---|---|
| 店舗公式サイト / Googleビジネス | ◎ 自動 | 定期クロール → HTML/構造化データをパース |
| X（旧Twitter） | △ 条件付き | API有料枠 or 公開タイムラインの限定取得。店舗アカウントを購読 |
| Instagram | △ 困難 | ログイン壁＋規約。公開投稿の範囲で限定的に |
| **公式LINE 配信** | ✕ **不可** | 他店のブロードキャストを外部取得する公開手段は存在しない |

### 半自動パイプライン（LINE / IG 等の埋め合わせ）

完全自動が無理なソースは、**告知テキストを貼る → LLMで正規化 → `data.js` に追記**で工数を最小化する。

```
[店舗のLINE/IG告知テキストを貼付]
      │
      ▼
[LLM抽出]  date / start / name / buyin / addon / stack / guarantee / reentry / tags
      │  （下記スキーマに正規化）
      ▼
[レビュー(任意) → data.js へ upsert] → サイト反映
```

各トーナメントは `source`（`auto`/`semi`/`manual`）と `verified` を持ち、UIで取得元をドット表示する。
未確認データは「(未確認)」と明示し、信頼性を担保する。

### Tournament スキーマ

```js
{
  id, venueId, name,
  date: 'YYYY-MM-DD', start: 'HH:MM',
  buyin: 円, addon: 円|null, stack: 点,
  guarantee: 円|null, reentry: true|'late'|false,
  prize: 文字列|null, tags: [],
  source: 'auto'|'semi'|'manual', verified: boolean
}
```

### タグ「大型」の定義

以下の**両方**を満たす場合のみ付与する。1店舗だけで完結する通常営業のトーナメント（各店舗のDAY1サテライト単体を含む）は、大きい大会の予選であっても対象外。

1. 次のいずれか
   - 複数店舗を巻き込んでサテライトが行われている**本線（決勝/本戦）**のトーナメント
   - 大会場を借りて実施する**イベント的な**トーナメント
2. 想定エントリー数が**最低100人以上**

## 法務・信頼性メモ

- フッターでは「当サイトは店舗・主催者が公開している情報を**集約する媒体**であり、賭博行為の勧誘・仲介を行うものではない」「賞品・プライズの取扱いは各店舗・主催者の規定による」と表明する。
  - 2026-07-22に、従来の「当サイトはアミューズメント(賭博性のない娯楽)目的の情報を掲載します」から変更した。
    旧文は**第三者である店舗・主催者のイベントの性質まで当社が保証する断定**になっており、賞金保証のある大型イベント(JOPT等)を掲載すると自社の表記と自社のコンテンツが矛盾するため。
    当社が保証できるのは「当社の立場」までであり、そこに限定する表現に改めた。
- **この文言を掲げる限り、参加受付・予約・エントリー代行・成果報酬型の送客は導入しない。** 導入を検討する場合は、先にフッター表記を見直しレビュー部の確認を通すこと。
  (旧文言が問題になったのは「書いた時点では正しかったが、事業の実態が動いて矛盾した」ため。同じ構造の再発を防ぐ。
  なお定額の店舗PR掲載枠・LINE配信は「広告」であって仲介ではないため、現行の文言と矛盾しない)
- 大型イベント特集ページには、非公式のまとめである旨・情報取得時点・当社の転記誤りの可能性を明示する免責を置く。
  賞金額を掲載する場合は、主催者発表である旨と、主催者側の建て付け(JOPTの場合「プライズは賞金ではなく選手契約」)を併記し、当社が言い換えない。
- 掲載は各店舗・主催者の公開情報ベース。参加前に公式確認を促す注意書きを常設。
- `data.js` は実店舗・実データで運用中(2026-07-14〜)。29店舗・300件超のトーナメント日程を掲載。
